Add / edit connectors

Setup guide

Select a connector kind for the config template, required fields, and where to get the API key. You can add multiple connectors of the same kind (e.g. two GitHub connectors for different repos) — each connector row is independent and can be selected per-run.

GitHub Issues (REST API v3) — use as an issue source, or to post spec comments back to issues.

Config JSON
{"repo": "owner/name"}
FieldRequired?Description
repo required Repository in owner/name format — e.g. makeplane/plane.
Secret (API token) GitHub Personal Access Token (PAT).
Needs repo scope for private repos; public_repo for public repos.
Generate at GitHub → Settings → Developer settings → Personal access tokens.
Base URL Leave blank for github.com.
For GitHub Enterprise Server set the API root, e.g.
https://github.your-org.com/api/v3
Multiple repos? Add one connector row per repo — e.g. GitHub – makeplane/plane and GitHub – my-org/service — each with a different {"repo": "..."}. When starting a run, pick the right connector from the "Issue source" dropdown.

Jira Cloud (REST API v3) — use as an issue source and/or a ticket sink for PM-generated stories.

Config JSON
{ "email": "you@example.com", "project_key": "ENG", "issue_type": "Story" }
FieldRequired?Description
email required Your Atlassian account email address.
project_key required Jira project key — the short prefix on all issue keys, e.g. ENG, PROJ. Tickets are created here.
issue_type optional Default issue type for stories. Defaults to Story.
spike_issue_type optional Issue type used for spikes / technical tasks. Defaults to Task.
Secret (API token) Jira API token (not your password).
Generate at Atlassian → Account settings → Security → API tokens:
https://id.atlassian.com/manage-profile/security/api-tokens
Base URL (required) Your Atlassian site URL — without a trailing slash.
Example: https://your-domain.atlassian.net
Found in the browser address bar when logged into Jira.

Plane (REST API) — use as an issue source and/or a ticket sink for PM-generated stories.

Config JSON
{ "workspace_slug": "acme", "project_id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx" }
FieldRequired?Description
workspace_slug required Your Plane workspace slug — visible in the URL after app.plane.so/, e.g. acme.
project_id required Plane project UUID. Found in Project → Settings — copy the ID from the URL or the settings page.
Secret (API key) Plane API key.
Generate at Workspace Settings → API Tokens (or your profile settings).
Self-hosted Plane: same path, just on your own domain.
Base URL Leave blank for Plane Cloud (https://api.plane.so).
For self-hosted Plane, set your API root, e.g.
https://plane.your-org.com
